~8 min
Open your project folder
In VS Code, always open a folder, not individual files. This tells VS Code "this is my entire project." Click File → Open Folder, navigate to your python-course folder on your Desktop, and click Select Folder. If VS Code asks "Do you trust the authors?" click Yes, I trust the authors. Your folder now appears in the Explorer panel on the left.
~3 min Why open a folder instead of a file?
Opening a folder lets VS Code understand your whole project at once. It can find related files, run the right Python interpreter, and give you better autocomplete suggestions. Opening a single file is like reading a book one random page at a time — VS Code works much better with the full picture.
Create and edit a Python file
In the Explorer panel, hover over your folder name — a small toolbar appears. Click the New File icon (a page with a + symbol). Type practice.py and press Enter. It opens automatically in the editor. Type print("hello") and press Ctrl+S to save. Watch the dot (●) on the file tab disappear when you save.
~4 min What does the dot on the tab mean?
The dot (●) means you have unsaved changes. Python can only run the saved version of your file. If you run a file with unsaved changes, it runs the old saved version — not what you just typed. Make pressing Ctrl+S a reflex: write a few lines, save, run. Build that habit now.
Run a Python file — two ways
With practice.py open and saved, try both methods:
Method 1 — Play button (easiest): Look for the triangle ▶ in the top-right corner of VS Code. Click it. A terminal panel opens at the bottom and your output appears. This is the best method while learning.
Method 2 — Terminal: Press Ctrl+` (the backtick key — top-left of keyboard, under Escape) to open the terminal. On Windows type python practice.py and press Enter. On Mac type python3 practice.py.
~5 min Which method should I use?
Use the play button for quick runs while learning — it's one click and works immediately. Use the terminal as you progress to more advanced work. Both produce the same result. The terminal is worth knowing because it's how professional developers run code, and you'll rely on it more as the course continues.
Three shortcuts worth memorizing now
These three keyboard shortcuts will save you time every single day. Practice each one right now while VS Code is open.
Ctrl+S — Save the current file. Do this constantly. Every time before you run.
Ctrl+Z — Undo the last thing you typed. Works up to hundreds of steps back.
Ctrl+` — Open or close the terminal. The backtick key is top-left, under Escape.

Install the Python extension — final setup step
Right now VS Code treats your Python files like plain text — it has no idea what the code means. Installing the Python extension gives VS Code the ability to understand Python: it highlights keywords in color, underlines mistakes as you type, offers autocomplete, and enables the play button to run your code. This is the last setup step before you start writing real programs.
Step 1. Click the Extensions icon in the activity bar — it looks like four squares on the left sidebar
Step 2. In the search box that appears, type Python
Step 3. Find the result named "Python" with publisher Microsoft — it will show tens of millions of installs
Step 4. Click the blue Install button
Step 5. Wait about 30 seconds. If VS Code shows a "Reload Required" button, click it.
